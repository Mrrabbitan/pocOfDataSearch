# 每周提交计划

## 目标
- 每周至少 5 次真实提交
- 每次提交聚焦单一小改动
- 记录改动范围与影响，便于复盘

## 本周拆分建议
1. 文档与说明更新
2. 测试或示例补充
3. 提示词模板优化
4. Web 或 CLI 体验微改动
5. 配置或核心逻辑微调

## 提交流程
- 变更前明确最小范围
- 变更后进行快速自检（必要时运行测试）
- 提交信息使用短前缀，例如：`docs:`、`test:`、`feat:`、`fix:`、`chore:`
